API documentation

User

GET

To get information about user you need to send GET request in format like this
/api/v0.x.x/user/?API_KEY=your_api_key&user_id=your_user_id
API_KEY and user_id are necessary
And you will get code 200 with info about user(role, username, id)
You will get 404 if User doesn't exist or 401 if API key incorrect

POST

You,also can creare user object using API, on POST query you dont need to pass API_KEY, but you have to pass username and password Format /api/v0.x.x/user/?username=username_you_want&password=password_to_account

Project

GET

To get information about project you need to send GET request in format like this
/api/v0.x.x/project/?API_KEY=your_api_key&project=your_project_id
API_KEY and project_id are necessary
And you will get code 200 with info about project(project_name, short_project_tag, id, 'priorities', 'subsystems', 'root', 'description')
You will get 400 if you didnt pass API key, 404 if Project doesn't exist, 401 if API key incorrect, 403 if you don't have access to project

POST

You can create project using API POST method
Format: /api/v0.x.x/user/?project_name=project_name_you_want&description=description_you_want&short_tag=short_tag_you_want(not necessary)

Issue

GET

To get information about issue you need to send GET request in format like this
/api/v0.x.x/issue/?API_KEY=your_api_key&tag=issue_tag
API_KEY and tag are necessary
And you will get code 200 with info about project('tracking', 'priority', 'state', 'description', 'steps_to_reproduce', 'summary', 'project_name', 'assign_on')
You will get 400 if you didnt pass API key, 404 if Project doesn't exist, 401 if API key incorrect, 403 if you don't have access to project

POST

You can create Issue using API POST method
Format: /api/v0.x.x/issue/?project_id=id_of_project&summary=base_info_about_issue&steps_to_reproduce=steps_to_reproduce_issue&description=issue_description&state=state_of_issue&priority=issue_priority
